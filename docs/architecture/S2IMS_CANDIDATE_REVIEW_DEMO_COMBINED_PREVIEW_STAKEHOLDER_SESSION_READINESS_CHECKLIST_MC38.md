# S²IMS Candidate Review Demo Combined Preview Stakeholder Session Readiness Checklist MC38

## Purpose

This checklist confirms readiness before conducting a real stakeholder feedback session for the hardened combined demo route.

The session is for feedback only. It must not collect approval, sign-off, AP-10B evidence, authority verification, production readiness approval, official evidence, scholarship decision, or assignment decision.

## Pre-Session Readiness

- [ ] MC37 dry run completed.
- [ ] MC37 readiness scorecard result is Proceed, or all Partial items have documented follow-up.
- [ ] Session invitation says feedback-only.
- [ ] Session agenda includes non-approval boundary.
- [ ] Stakeholder categories are role-based, not named as approval owners.
- [ ] AP-10B separation statement is prepared.

## Facilitator Readiness

- [ ] Facilitator has MC36 walkthrough script open.
- [ ] Facilitator has MC36 Q&A guardrails open.
- [ ] Facilitator can explain both sections without implying approval.
- [ ] Facilitator can answer production-readiness questions safely.
- [ ] Facilitator can redirect AP-10B questions to separate governance process.
- [ ] Facilitator will not ask for approval, sign-off, readiness confirmation, or authority verification.

## Privacy Readiness

- [ ] Privacy reminder prepared.
- [ ] No real student data is used.
- [ ] No real personnel data is used.
- [ ] No real stakeholder data is displayed.
- [ ] Note-taker understands PII must not be recorded.
- [ ] If a participant mentions PII, facilitator will stop, redirect, and instruct note-taker not to record it.

## Route Readiness

- [ ] `/admin/candidate-review-demo` returns 200 OK.
- [ ] Demo route remains hidden from sidebar navigation.
- [ ] Demo route remains hidden from topbar navigation.
- [ ] Demo route remains hidden from mobile navigation.
- [ ] Candidate review diagnostic preview is visible.
- [ ] Feedback backlog preview is visible.
- [ ] Route-level warning copy is visible.
- [ ] No feedback form is present.
- [ ] No save/submit/approve/assign/export/notify controls are present.

## Note-Taking Readiness

- [ ] MC38 note template is available.
- [ ] Note-taker records reviewer category only.
- [ ] Note-taker records anonymized feedback themes only.
- [ ] Note-taker does not record names, contact data, IDs, signatures, approval wording, or official evidence claims.
- [ ] Note-taker marks governance-sensitive feedback separately.
- [ ] Note-taker confirms `nonApprovalConfirmed: true`.

## Q&A Readiness

- [ ] Production-readiness answer prepared.
- [ ] Official-evidence answer prepared.
- [ ] AP-10B approval answer prepared.
- [ ] Persistence/storage answer prepared.
- [ ] Audit-write answer prepared.
- [ ] Assignment/approval/scholarship decision answer prepared.
- [ ] Legal/DPO sign-off answer prepared.

## Post-Session Readiness

- [ ] Synthesis owner identified.
- [ ] Synthesis categories prepared.
- [ ] Governance-sensitive items will be separated.
- [ ] Follow-up branch candidates will remain planning-only unless separately approved.
- [ ] No AP-10B gate update will be recorded.
- [ ] No official evidence will be created.
- [ ] No approval or sign-off record will be created.

## Stop Conditions

Stop or pause the session if:
- real PII is mentioned or shown
- any participant attempts to provide approval
- any participant attempts to sign off
- any participant attempts to treat the demo as official evidence
- any participant asks to update AP-10B status during the session
- any participant asks to make scholarship or assignment decisions
- facilitator cannot answer a governance-sensitive question safely

## AP-10B Separation Checklist

- [ ] State AP-10B owners: 0/7.
- [ ] State AP-10B approvals: 0/7.
- [ ] State AP-10B blockers: 9/9 active.
- [ ] State AP-10C remains blocked.
- [ ] State AP-11 remains blocked.
- [ ] State stakeholder feedback session does not change any AP-10B value.
- [ ] State AP-10B requires separate formal governance process.
