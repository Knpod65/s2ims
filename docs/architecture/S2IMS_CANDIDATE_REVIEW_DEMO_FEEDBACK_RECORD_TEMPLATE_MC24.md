# S²IMS Candidate Review Demo — Feedback Record Template (MC24)

## Purpose

This template provides a standardized, safe format for recording feedback from S²IMS candidate review demo walkthrough sessions. Using this template ensures that:

- Feedback is recorded in a structured, classifiable format
- No personal data (PII) is captured in feedback records
- No record is accidentally treated as approval or governance evidence
- All records include an explicit non-approval confirmation

Use this template alongside the master intake plan (`S2IMS_CANDIDATE_REVIEW_DEMO_FEEDBACK_INTAKE_PLAN_MC24.md`) and the classification matrix (`S2IMS_CANDIDATE_REVIEW_DEMO_FEEDBACK_CLASSIFICATION_MATRIX_MC24.md`).

---

## Safe Feedback Record Template

Copy this template for each piece of feedback from a session.

---

**FEEDBACK RECORD — S²IMS CANDIDATE REVIEW DEMO**

| Field | Value |
|-------|-------|
| **Record ID** | (sequential number within session, e.g., 2026-05-17-001) |
| **Session date** | (YYYY-MM-DD) |
| **Stakeholder group / category** | (Role or group only — e.g., "Admin staff group", "External advisor cohort") |
| **Demo facilitator** | (Role or name — internal use only) |
| **Feedback category** | (One of: UX Clarity / Copy-Content / Accessibility / Privacy-PDPA / Workflow Understanding / Training-Readiness / Risk-Concern / Future Enhancement / Out of Scope) |
| **Feedback summary** | (Brief, factual description — no personal opinions attributed to named individuals) |
| **Risk / concern noted** | (Yes / No — if Yes, describe briefly) |
| **Proposed follow-up** | (Docs-only action item type from the intake plan, or "None") |
| **Non-approval confirmation** | This record does not constitute approval of any kind. No AP-10B action was taken. No official evidence was created. |

---

## Non-Approval Confirmation Language

Every feedback record must include the following statement verbatim in the "Non-approval confirmation" field. This language must not be abbreviated, omitted, or paraphrased:

> "This record does not constitute approval of any kind. No AP-10B action was taken. No official evidence was created."

---

## Privacy Exclusion List

The following information must NOT appear anywhere in a feedback record:

- Full names of individual stakeholders, students, or personnel
- National ID numbers, passport numbers, or government-issued IDs
- Phone numbers or personal email addresses
- Home or work addresses
- Student application IDs or scholarship record IDs that correspond to real individuals
- Medical, health, or financial data
- Private hardship or personal circumstance details
- Any data that could identify a real individual

If any of the above is inadvertently included in a draft record, it must be removed before the record is filed.

---

## AP-10B Separation Statement

All feedback records from walkthrough sessions are explicitly separated from AP-10B governance:

- No feedback record constitutes an AP-10B approval
- No feedback record nominates an AP-10B owner
- No feedback record changes the AP-10B gate status
- The AP-10B gate remains: 0/7 owners, 9/9 blockers — unchanged
- AP-10C: Blocked — unchanged
- AP-11: Blocked — unchanged
