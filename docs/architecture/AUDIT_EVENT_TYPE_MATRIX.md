# Audit Event Type Matrix

This matrix proposes future audit event types for S2IMS. It is planning-only and does not implement event creation, validation, API calls, or persistence.

| Event type | Domain | Actor role | Target type | Reason required | Suggested min length | Severity | Privacy level | Persistence priority | Current UI surface | Current state | Future notes |
|---|---|---|---|---|---:|---|---|---|---|---|---|
| `staff.document.verify` | Staff documents | staff, admin | document/application | No | 0 | low | staff_internal | Medium | `DocumentVerificationPanel` | Callback + toast only | Persist event with document ID, application token, no raw file contents. |
| `staff.document.reject` | Staff documents | staff, admin | document/application | Yes | 20 | medium | staff_internal | High | `DocumentVerificationPanel` | Prototype-safe warning + non-empty reason | First Staff document action candidate for mock audit writer after AP-2. |
| `staff.document.request_replacement` | Staff documents | staff, admin | document/application | Yes | 20 | medium | staff_internal | High | `DocumentVerificationPanel` | Prototype-safe warning + non-empty message | Keep student-facing language supportive; staff reason stays operational. |
| `staff.disclosure.approve_identity_reveal` | Disclosure | staff, admin | disclosure request/candidate | Yes | 30 | critical | pii_restricted | Critical | `DisclosureApprovalModal` | Warning claims logged/irreversible; no reason input | Needs product decision before runtime migration. |
| `staff.disclosure.reject_identity_reveal` | Disclosure | staff, admin | disclosure request/candidate | Yes | 20 | medium | masked_candidate | High | `DisclosureRejectionModal` | 15-character reason; logged copy | Align min length and copy stage later. |
| `staff.match.override_decision` | Matching | staff, admin | match/application | Yes | 30 | high | staff_internal | High | `MatchOverrideModal` | 20-character reason; logged copy | Policy says high risk; review persistence truth before copy reuse. |
| `provider.shortlist.request` | Provider shortlist | provider | shortlist request/candidate tokens | Yes | 20 | medium | provider_token_only | Medium | `ShortlistRequestModal` | 10-character reason; privacy warning | Must never expose raw student identity. |
| `provider.shortlist.submit_reason` | Provider shortlist | provider | shortlist request | Yes | 20 | medium | provider_token_only | Medium | `ShortlistReasonField` in modal | Reason field only | Could share event with request or separate draft/submit events later. |
| `admin.role.assign` | Admin access | admin | user/role assignment | Yes | 40 | critical | admin_sensitive | Critical | `RoleAssignmentPanel` | Component-level 20-character validation | Confirm route wiring and persistence before official copy. |
| `admin.role.remove` | Admin access | admin | user/role assignment | Yes | 40 | critical | admin_sensitive | Critical | `RoleAssignmentPanel` | Component-level 20-character validation | Should include before/after role set in allowlisted metadata. |
| `admin.export.generate` | Export | admin, staff, esq where allowed | export job/report | Yes for sensitive exports | 30 | high | export_sensitive | High | `admin/export`, `admin/audit-log` | Client-side download, no reason | Export should create its own audit event once persistence exists. |
| `admin.permission.change` | Admin access | admin | permission/scope | Yes | 40 | critical | admin_sensitive | Critical | No dedicated current page confirmed | Config concept only | Needs product/route decision before implementation. |
| `system.ocr.process_document` | OCR | system | OCR job/document | No | 0 | medium | staff_internal | Medium | `staff/ocr` | Mock/process UI | Store job ID, outcome, confidence, no document contents. |
| `system.data_quality.flag` | Data quality | system, staff | data quality issue | Optional | 0 | medium | operational_internal | Medium | `staff/data-quality` | Mock issue states | Staff-created corrections may need a separate manual correction event. |
| `system.integration.sync_failed` | Integrations | system | sync job/integration | No | 0 | medium | operational_internal | Medium | Data freshness indicators | Status display only | Useful for admin/staff system health later. |
| `admin.audit.export` | Audit viewer/export | admin | audit export | Yes | 30 | high | admin_sensitive | High | `admin/audit-log` export button | Mock CSV export | Should record export scope, row count, field allowlist version. |
| `staff.identity.reveal_application` | Staff application detail | staff, admin | application/student token | Yes | 30 | high | pii_restricted | Critical | Staff application reveal modal | Mock toast; logged copy | Existing copy implies logging; align with persistence stage. |
| `staff.note.add` | Staff operations | staff | application | Optional | 0 | low | staff_internal | Low | Staff application detail | Toast only | May remain operational log rather than high-signal audit event. |
| `staff.application.status_change` | Staff applications | staff | application | Optional/required by transition | 20 | medium | staff_internal | Medium | Staff application detail | Status state + toast | Sensitive transitions may require reason later. |
| `system.manual_data_correction` | Data correction | staff, admin, system | operational record | Yes | 30 | high | admin_sensitive | High | OCR/save/data quality surfaces | Config concept only | Must store source and correction reason, not full raw payload. |
