## Prohibited Copy

Do not use any of the following in Admin audit surfaces at Stage 1 (mock persistence):

### Persistence implications
- "Persisted", "Stored", "Saved", "Recorded in system"
- "Official record", "System of record", "Permanent record", "Archived"
- "Immutable", "Tamper-proof"

### Evidence/validation implications
- "Verified", "Confirmed", "Validated", "Certified"
- "Legally binding", "Compliant with", "Meets requirements"
- "Logged and auditable"

### Action completion claims
- "Action completed", "Process finalized", "Workflow concluded"

### Data integrity claims
- "Data integrity verified", "Information accurate", "Details confirmed"
- "Record is complete", "No missing information"

### Escalation/urgency words that overstate mock events
- "Critical failure logged" — wrong because events are mock
- "Audit trail updated" — implies real backend persistence

## Thai/English Copy Guidance

### General principles

1. **English is primary** for all Admin audit surfaces. Thai is additive.
2. Technical terms may remain in English when no clear Thai equivalent exists: "mock", "demo", "metadata", "token", "API".
3. "Audit" remains in English in Thai copy (standard Thai practice in official/regulatory contexts).
4. "Stage 1 / mock-only" terminology should use the closest Thai equivalent: "เดโม" (demo) rather than "จำลอง" (simulated) for consistency with existing copy.

### Terms that lack Thai translations in current code

These source labels currently output English for both `lang==='th'` and `lang==='en'`:

| Label | Thai translation needed |
|-------|------------------------|
| "Fixture mock" | "เดโม (ฟิกซ์เจอร์)" or "บันทึกเดโม" |
| "Writer mock" | "เดโม (สร้างขึ้น)" or "บันทึกเดโม" |
| "Writer mock" (drawer source badge) | Same as above |
| "Fixture mock" (drawer source badge) | Same as above |

### Copy parity check

Before releasing any Polish pass, confirm every new English string has a Thai equivalent and vice versa in `page.tsx` and `AdminAuditEventDetailDrawer.tsx`.
